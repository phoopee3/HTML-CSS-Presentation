HTML

  • Is everyone familiar with HTML?
  • What is HTML?

    • HTML is Structure

    
<html>
	<head>
		My Web Site!
	</head>
	<body>
		
    Hello World
    
		
    Welcome to my web page!
    
	</body>
</html>
    The most important thing to take away from this presentation is that there should be a separation of structure and style.

    Tags help structure

    • nav
    • header
    • section
    • aside
    • p
    • div
    • table thead th tbody tr td
    • ul li, ol li
    • img
    • hr
    • ... and many more
    These are obviously just a few of the HTML tags available

    Tags can have attributes
    An attribute is a modifier to the HTML element. It helps modify the default behavior of the tag, like telling an a tag where to link to, or what class an element should use for styling.

    Tags can have a Class and/or an Id
    Classes and IDs lead us into the next topic, CSS

    CSS

  • Is everyone familiar with CSS?
  • What is CSS?

    • What is CSS?

    CSS is Style

    Cascading style sheets (CSS) are a way to add style to your web page.

    Again, the most important thing to take away from this presentation is that there should be a separation of structure and style.

    Syntax

    
selector {
	property : value;
}
  • The selector is the element that you are trying to match
  • The property is what we want to change, color, font, margin, etc.

    • Styling a tag

    
p {
	color : #333333;
	font-size : 12px;
}
  • This will style all p tags on the site
  • This way is a little too generic

    • Styling a Tag

    Styling a tag can be a good way to cover a lot of your site, but it will limit the amount of detail you can apply to the site.

    Make a class instead

    Classes allow you to reuse the styling on multiple elements.

    
.call-to-action {
	background-color : #ff0000; /* red */
	color : #ffffff; /* white */
	font-weight : bold;
	font-size : 14px;
}

    And then in the html

    
This is attention getting!

    This might be a bit harsh on the eyes if overused.

    Naming your classes is HARD, but I will stress that you should use semantic names, IE names that are descriptive about what the class represents without specifying the content they enclose.

    An example is a class like "red-header-text", instead using something like "call-to-action".

    There are a few methods for naming classes, like BEM, SMACSS, and OOCSS, see the links section for more information

    Specificity

    These selectors have the same specificity, so they are applied in the order they are listed,
    so all p tags will be blue

    
p { color : red; }
p { color : blue; }
    on jsbin.com

    Specificity...

    While the blue rule comes last, the first rule is more specific, so both rules will be applied appropriately

    
div p { color : red; }

    Chrome DevTools

    A Developers Best Friend!
    Or just someone who's curious

    What is Chrome DevTools?

    Chrome DevTools is a set of authoring, debugging, and profiling tools built into Google Chrome.

    Similar tools are available in FireFox and Safari, and a crappy version in IE, that allow the developer insight into how a page is created.

    DevTools

    The main set of tools that we will focus on are:

    • Elements - The rendered source code of the page
    • Console - An interactive console where you use to debug the site
    • Sources - A list of files that the page is calling
    • Network - Allows you to view what information is passing over the network

    More Information

  • Console - An interactive console where you can type commands (Javascript) to debug/manipulate the page
  • Sources - Includes CSS, JS, images
  • Network - Shows HTML, CSS, image, and network calls, API calls where you can see the headers sent and data returned

    • Elements

    Tweak the layout and design of your site by freely manipulating the DOM and CSS.

    Console

    Log diagnostic information during development or interact with the JavaScript on the page.

    Sources

    Debug your JavaScript using breakpoints or connect your local files via Workspaces to use DevTools as a code editor.

    Network

    Optimize page load performance and debug request issues.

    Emmet

  • Who has heard of Emmet?
  • Programmers are lazy

    • Emmet is...

    Emmet is a plugin for many popular text editors which greatly improves HTML & CSS workflow.

    At it's most basic, emmet allows you to write shorthand code that gets expanded to full blown html, which you can then tab through and fill out as necessary.

    Emmet.io

    Syntax

    • Elements - like p, div, h2, ul, etc
    • Nesting operators
      • > Child - This will nest elements
      • + Sibling - This will put elements adjacent to each other
      • ^ Climb up - Go up a level, break out of the nesting

    Syntax

    • * Multiplication - Repeat the element X times
    • () Grouping
    • . or # Class or ID - Give an element a class or ID
    • [] Attributes - Set attributes in an element
    • {} Content - Set content in an element
    • $ Numbering - Set a number in a multiplication loop

    Example

    This is a bit of a contrived example, but it shows a bunch of examples from the previous slides:

    
	html>head>title{My Site}^body>h1{Welcome!}+ul>li*3>a{Item $}^^button.call-to-action[data-foo="bar"]{Click Me}

    Example

    Emmet will expant that to:

    
	<html>
	<head>
		<title>My Site</title>
	</head>
	<body>
		<h1>Welcome!</h1>
		<ul>
			<li><a href="">Item 1</a></li>
			<li><a href="">Item 2</a></li>
			<li><a href="">Item 3</a></li>

    SASS

  • What is SASS?
  • Who has heard of SASS, Compass, or LESS?

    • SASS is CSS (and more)

    SASS is an extension of CSS3 which adds nested rules, variables, mixins, selector inheritance, and more.

    SASS generates well formatted CSS and makes your stylesheets easier to organize and maintain.

    More information

    How is SCSS like CSS?

    CSS

    
.call-to-action {
	background-color : red;
	color : white;
	font-weight : bold;
}

    SCSS

    
.call-to-action {
	background-color : red;
	color : white;
	font-weight : bold;
}
    You might think this looks exactly the same, and you'd be right. You can write straight CSS in SASS, so what does SASS give us?

    Advantages

    • Variables
    • Nesting
    • Mixins
    • Functions
    • Compiling

    Advantages - Variables

    SCSS

    
$alert-color : #ffffff;
$alert-background-color : #ff0000;
$font-size : 13px;

.call-to-action {
	background-color : $alert-background-color;
	color : $alert-color;
	font-size : $font-size;
}

    Advantages - Variables

    Converts to CSS

    
.call-to-action {
	background-color : #ff0000;
	color : #ffffff;
	font-size : 13px;
}

    Advantages - Nesting

    In CSS, you have to explictly write out each selector, even if they share the same base tag.

    
nav ul {
	margin : 0;
	list-style : none;
}
nav ul li {
	display : inline-block;
}
nav a {
	display : block;
	padding : 6px 12px;
	text-decoration : none;
}

    Advantages - Nesting

    In SASS you can nest your CSS selectors.

    
nav {
	ul {
		margin : 0;
		list-style : none;
		li {
			display : inline-block;
		}
	}

	a {
		display : block;
		padding : 6px 12px;
		text-decoration : none;
	}
}

    Advantages - Mixins

    Mixins are methods that you can call to help you write less code

    SCSS

    
@mixin border-radius ($radius) {
	-webkit-border-radius: $radius;
	-moz-border-radius:    $radius;
	-ms-border-radius:     $radius;
	border-radius:         $radius;
}

.box {
	@include border-radius(10px);
}

    Advantages - Mixins

    Compiles to CSS

    
.box {
	-webkit-border-radius: 10px;
	-moz-border-radius:    10px;
	-ms-border-radius:     10px;
	border-radius:         10px;
}

    Advantages - Functions

    Many built in functions, but you can also write your own

    
p {
	color : hsl(0, 100%, 50%);
}	
						
    
p {
	color : #ff0000;
}

    Advantages - Compiling

    SASS can compile to different levels of compression

    • Nested
    • Expanded
    • Compact
    • Compressed
